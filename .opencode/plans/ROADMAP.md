# Tafiles — 未来路线图规划

> 本文档基于现有功能体系，深入思考 Tafiles 作为系统级桌面应用的可扩展方向。

---

## 一、现状总结

Tafiles 目前是一个**系统级文件预览与图片编辑工具**，核心能力：

| 领域 | 能力 |
|------|------|
| 文件浏览 | 树状目录 + 多视图（大/中/小/列表）+ 面包屑导航 + 窗口 resize |
| 图片预览 | 全屏无边框沉浸式预览 + 半透明遮罩 + 缩放/旋转 |
| 图片编辑 | 矩形/箭头/文字标注 + 马赛克 + 裁切 + 撤销/重做 + 保存/另存 |
| 文档预览 | PDF / Excel / Word / 纯文本 / 代码（语法高亮） |
| 系统集成 | Windows 原生图标（SHGetImageList）+ 默认应用打开 + 右键菜单禁用 |
| 多窗口 | 独立预览窗口 + 窗口位置/大小记忆 + 全屏无边框 |
| 技术栈 | Vue3 + Vite + Element Plus + Tauri 2.x (Rust) |

---

## 二、未来发展方向

### 方向 A：通用文件管理器（资源管理器增强）

**核心思路：** 从「图片浏览工具」进化为「通用文件管理器」。

#### A1. 文件操作基础
- [ ] 文件/文件夹的创建、重命名、删除、移动、复制（右键菜单 + 快捷键）
- [ ] 多选（Ctrl/Shift + 点击，框选）
- [ ] 拖拽操作（拖拽移动/复制）
- [ ] 剪贴板集成（Ctrl+C/V/X + 系统剪贴板互通）
- [ ] 文件搜索/过滤（按名称、类型、日期、大小）
- [ ] 文件属性面板

#### A2. 文件列表增强
- [ ] 排序（点击列头排序）
- [ ] 分组（按类型/日期等）
- [ ] 筛选器（只显示图片/文档等）
- [ ] 双面板/分栏模式

#### A3. 标签页
- [ ] 多标签页浏览

#### A4. 联网文件系统
- [ ] WebDAV 协议支持
- [ ] SMB/Samba 网络共享
- [ ] FTP/SFTP 支持
- [ ] 云存储接入（OSS / S3 / COS）
- [ ] 挂载为虚拟文件系统

### 方向 B：插件化文件预览引擎

**核心思路：** 设计插件化架构，第三方可扩展新文件类型预览。

#### B1. 架构设计
- 内置预览器: Image / Office / Code / Pdf
- 外挂插件: Archive / Video / Audio / CAD / Hex 等
- 插件系统: 接口规范 + 热加载 + 插件市场 + 沙箱

#### B2. 可优先实现的扩展预览器
- [ ] ZIP/RAR/7z 预览 + 解压
- [ ] 视频缩略图 + 播放
- [ ] 音频波形 + 播放
- [ ] Markdown WYSIWYG 渲染预览
- [ ] HEIC/RAW 图片格式支持
- [ ] CAD/DWG → SVG 预览
- [ ] Font 字体预览
- [ ] SQLite/CSV 表格预览

### 方向 C：WebView2 内核浏览器

**核心思路：** Tauri 底层为 WebView2，Tafiles 天然可作为轻量级浏览器。

- [ ] 地址栏 + 前进/后退/刷新
- [ ] 多标签页管理
- [ ] 书签/下载/历史管理
- [ ] 阅读模式 / 广告拦截 / 用户脚本
- [ ] 网页资源嗅探下载
- [ ] 页面翻译
- [ ] 内置 DevTools

### 方向 D：AI 集成

**核心思路：** 接入 AI API，围绕文件服务打造智能助理。

- [ ] 多模型适配（OpenAI / Claude / Gemini / Ollama 本地）
- [ ] 图片分析/描述/增强
- [ ] 智能抠图 / OCR
- [ ] AI 批量重命名/分类
- [ ] 文档总结/翻译
- [ ] 自然语言搜索
- [ ] API Key 管理面板 + 消费统计

### 方向 E：系统工具 + 效率增强

**核心思路：** 利用 Rust 后端能力，提供实用系统工具。

- [ ] 批量重命名（正则/序号/替换）
- [ ] 批量格式转换（图片互转/压缩）
- [ ] 批量 PDF 合并/拆分
- [ ] 快速启动器（类似 Wox / uTools）
- [ ] 剪贴板历史管理器
- [ ] 屏幕截图 + 标注
- [ ] 文件哈希校验
- [ ] 磁盘空间分析器
- [ ] 重复文件查找
- [ ] 图片 EXIF 查看/编辑/清除
- [ ] 颜色选取器
- [ ] 二维码生成/识别
- [ ] HTTP 文件服务（局域网共享）
- [ ] P2P 文件传输（WebRTC）

### 方向 F：file-service 统一文件服务

**核心思路：** 统一浏览本地/网盘/远程文件。

- 文件源注册表：本地 / 网盘 API / WebDAV / SMB / SFTP / S3
- 虚拟源：最近文件 / 收藏 / 搜索结果 / 压缩包虚拟 FS
- 文件搜索：全文搜索 + 语法搜索（type:pdf size:>10mb）
- REST API 模式：手机浏览器远程访问文件
- 文件上传（拖拽上传到本机）
- WebSocket 实时同步

## 三、分阶段实施建议

### Phase 1：夯实基础（1-2 个月）
1. 文件操作 CRUD（创建/重命名/删除/移动/复制）
2. 文件搜索/过滤
3. 多选操作
4. 右键菜单
5. 标签页切换

### Phase 2：预览扩展 + 插件化（2-3 个月）
1. 压缩包浏览 + 解压
2. 插件预览器接口规范
3. 视频/音频预览
4. Markdown 渲染预览
5. HEIC/RAW 支持

### Phase 3：系统工具 + AI 集成（3-6 个月）
1. 批量重命名
2. EXIF 查看/编辑
3. AI API 接入框架
4. AI 图片分析
5. 屏幕截图

### Phase 4：file-service + 浏览器 + 网盘（6-12 个月）
1. 统一文件抽象层
2. WebDAV 客户端
3. 基础浏览器
4. 网盘 API 接入
5. 跨设备文件服务

## 四、技术可行性评估

| 方向 | 可行性 | 关键挑战 |
|------|--------|---------|
| 文件操作 CRUD | ✅ 简单 | Rust std::fs 封装 |
| 压缩包预览 | ✅ 简单 | zip/tar crate，只读模式 |
| 插件系统 | ⚠️ 中等 | 热加载沙箱设计 |
| 视频播放 | ⚠️ 中等 | FFmpeg 绑定或 WebCodecs |
| 网盘挂载 | ⚠️ 中等 | OAuth2 + REST API |
| 浏览器标签页 | ✅ 简单 | Tauri 多窗口 |
| AI 集成 | ⚠️ 中等 | HTTP 调用 API，UI/UX 设计 |
| HTTP 文件服务 | ⚠️ 中等 | Rust HTTP server |
| 系统拖拽 | ⚠️ 复杂 | Shell API / Tauri 插件 |

## 五、核心架构演进

### 当前架构
Vue3 (前端) ← invoke → Tauri (Rust 后端) → 文件系统 / Win32 API

### 目标架构
Vue3 (前端) ← invoke → Tauri (Rust 后端) → 文件系统 / Win32 API
                  ↑                            ↓
            插件系统                   文件源抽象层
                  ↑                            ↓
            AI 引擎适配器              HTTP 文件服务 / 浏览器标签页
